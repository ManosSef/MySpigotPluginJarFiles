BUILD 2

Additions
- Added success messages to /rolltasks (OTHER-211)
- Added /gift (OTHER-212)
	- Gifts a heart to another player, once per session
	- Syntax: /gift <player>
- Added /task, replacing /succeed, /reroll and /fail (OTHER-213)
	- Succeeds, rerolls, or fails the player's task
	- Syntax: /task (succeed|reroll|fail)

Changes
- The plugin now logs the number of each type of tasks in the config as well as the number of players with each type of forced tasks
- Extra rewards from succeeding tasks now spawn at the player's location if the plugin's item spawn location has not been set
- Players' maximum health no longer decreases when the damage was taken during invulnerability ticks
- Changes to the plugin's YML files are no longer overwritten when the plugin is disabled
- The delays between the titles shown when tasks are rolled are now shorter (OTHER-202)
- The delay between succeeding or failing a red task and a new one being rolled is now longer (OTHER-218)

Fixes
- OTHER-200 - The plugin does not depend on SemiHardcore
- OTHER-201 - The plugin does not create a config.yml file when one doesn't exist
- OTHER-203 - Taking damage doesn't always reduce the player's maximum health
- OTHER-204 - Books given by the plugin have no data
- OTHER-205 - The arguments of /edithealth tab complete strings that don't start with the typed argument
- OTHER-206 - Succeeding a task with enough health that you would get item rewards causes an error
- OTHER-207 - Books are not removed when succeeding, rerolling or failing a task
- OTHER-208 - Failing a task that removes health while having less than that amount of health causes an error
- OTHER-209 - Succeeding a task that would increase your health to more than the limit still shows the full number of hearts in the title
- OTHER-210 - The maximum health is not reset upon death
- OTHER-214 - Getting down to 1 life with no task doesn't roll a task
- OTHER-215 - Being revived into having 1 life doesn't roll a task
- OTHER-216 - Gaining a life while having a red task and succeeding or failing rolls a new task
- OTHER-217 - Getting down to 1 life with an easy or hard task and succeeding or failing doesn't roll a task
